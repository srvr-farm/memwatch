# Memwatch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Rust TUI memory monitor similar to `cpuwatch`, showing system memory, DIMM speed, Intel IMC bandwidth, and top resident-memory processes.

**Architecture:** The crate mirrors `cpuwatch`: a small terminal app with focused modules for CLI, memory parsing, DMI parsing, Intel bandwidth counters, process scanning, snapshot aggregation, rendering, and app lifecycle. Hardware-facing modules accept injectable filesystem roots where practical, so parsing and calculations are covered by fixture tests while live perf/DMI failures become diagnostics.

**Tech Stack:** Rust 2021, `anyhow`, `clap`, `humantime`, `ratatui`, `crossterm`, `libc`, `tempfile` for tests.

---

## File Structure

- `Cargo.toml`: crate metadata and dependencies.
- `Makefile`: build, install, capability, test, lint, and clean targets matching `cpuwatch`.
- `.gitignore`: Rust build output ignores.
- `src/main.rs`: small delegating binary entry point.
- `src/lib.rs`: public `run()`, mode selection, TUI lifecycle, and event loop.
- `src/cli.rs`: command-line parsing and duration validation.
- `src/memory.rs`: `/proc/meminfo` parsing and memory summary calculations.
- `src/dmi.rs`: DMI/dmidecode-style memory device parsing and live collection.
- `src/bandwidth.rs`: Intel uncore IMC PMU event discovery, `perf_event_open` counter reader, and bandwidth math.
- `src/processes.rs`: `/proc/<pid>/status` scanning and top-RSS sorting.
- `src/snapshot.rs`: combines current data, previous bandwidth samples, and diagnostics.
- `src/render.rs`: plain text report and `ratatui` TUI layout.
